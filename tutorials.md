## 2. Core Module Deep-Dives & Flow Mechanics

### Android Storage Access Framework (SAF) & Native Persistence
To guarantee strict Android sandbox safety, Shepherd utilizes the **Storage Access Framework**.
* **Intent-Driven Authorization:** On the `OnboardingScreen`, users choose a root folder. By launching `ActivityResultContracts.OpenDocumentTree()`, the platform acquires a persistent URI via `contentResolver.takePersistableUriPermission`.
* **Sub-folders & File Resolving:** `SafFileManager` translates database file descriptors directly into concrete `DocumentFile` trees. This bypasses fragile hardcoded JVM `java.io.File` absolute paths, which are deprecated on modern Android.
* **Native Launch Integrations:** To open `.docx` files generated by POI, the app queries the framework path, packages it with explicit read intents (`Intent.FLAG_GRANT_READ_URI_PERMISSION`), and broadcasts an intent targeting office suite tools installed on the system (e.g., Google Docs, Microsoft Word).

### High-Fidelity Word Compilations (Apache POI Generator)
The `DocxGenerator` acts as a text compiler that parses stream-derived markdown syntax and formats a raw byte sequence containing native Word `.docx` file properties:
1. **Paragraph Mapping:** Splits blocks based on line breaks, removing markdown-specific characters.
2. **Dynamic Styling:** Translates structural signifiers (`#`, `##`, `-`, `*`, `>`) into document elements:
   * Main Headings are assigned font sizes, bold weights, and custom line spacing.
   * Quotations prefixed with `>` are formatted with structural indentation (`indentationLeft`) and standard italic styling.
   * Standard bullet lists are structured with precise indentation offsets, adding clean indentation styling.
3. **Stream Outputs:** Compiles POI buffers, outputting raw byte arrays written back directly to the authorized SAF location.

```kotlin
// Simplified compilation pipeline in DocxGenerator.kt
val document = XWPFDocument()
val paragraph = document.createParagraph()
if (trimmedLine.startsWith("# ")) {
    paragraph.alignment = ParagraphAlignment.CENTER
    val run = paragraph.createRun().apply {
        isBold = true
        fontSize = 20
        fontFamily = "Times New Roman"
    }
    run.setText(trimmedLine.substring(2))
}
```

### Tutorial A: Adding a New AI Formatting Mode & System Prompt
Suppose your team wants to add a **Devotional** formatter that converts raw notes into a 5-day daily theological study guide.

#### Step 1: Add the Enum Key in `/app/src/main/java/com/example/data/remote/GeminiService.kt`
Find the `FormatMode` enum definition (around Line 12) and append your new mode:
```kotlin
enum class FormatMode {
    SERMON, LETTER, REPORT, NOTES, GENERAL, DEVOTIONAL
}
```

#### Step 2: Implement the System Prompt in `GeminiService.kt`
Locate `getSystemPrompt` and define the format structure:
```kotlin
FormatMode.DEVOTIONAL -> """
    You are an expert biblical devotion writer. Format the input raw scripture or note into a 5-Day Devotional Guide.
    
    Format structure exactly as:
    # **DEVOTION TITLE** (infer from text)
    
    **DAY 1: [Theme Title]**
    - **Focus Scripture:** [Verse]
    - **Reflective Commentary:** (1 paragraph summarizing the theme)
    - **Daily Challenge / Prayer:** (1 short call-to-action application)
    
    [Repeat the exact structure for Day 2, Day 3, Day 4, and Day 5]
    
    Fix all grammar, preserve therapeutic tone, and return ONLY the markdown template outputs.
""".trimIndent()
```

#### Step 3: Map UI Selectors in `/app/src/main/java/com/example/presentation/screens/AiEditorScreen.kt`
To allow the pastor or writer to select "Devotional Mode", add a choice block within the formatting UI elements. Open `AiEditorScreen.kt`, examine how `FormatMode` selectors or chips are structured, and register `FormatMode.DEVOTIONAL`:

```kotlin
// Example format picker chip implementation in AiEditorScreen.kt
FormatMode.values().forEach { mode ->
    FilterChip(
        selected = (formatMode == mode),
        onClick = { viewModel.setAiGenerateMode(mode) },
        label = { 
            Text(
                text = when(mode) {
                    FormatMode.SERMON -> "Sermon Outline"
                    FormatMode.LETTER -> "Pastoral Letter"
                    FormatMode.REPORT -> "Ministry Report"
                    FormatMode.NOTES -> "Study Notes"
                    FormatMode.GENERAL -> "Polished Copy"
                    FormatMode.DEVOTIONAL -> "5-Day Devotional"
                }
            )
        }
    )
}
```

#### Step 4: Map Word Styles in `/app/src/main/java/com/example/data/local/DocxGenerator.kt`
Optionally, enhance the Word Document compiler to recognize Devotional layouts (e.g., giving "DAY X" headers a distinct style):
```kotlin

    paragraph.alignment = ParagraphAlignment.LEFT
```

### Tutorial B: Customizing UI Themes & Color Schemes
Shepherd includes custom, modern accent colors (such as ShepherdClassic Dark/Light palettes, Forest Green highlights, ShepherdGold, and Ivory). Here is how to register an entirely new **"Abbey Blue" (Monastic) color scheme**.

```kotlin
val AbbeyBlueSecondary = Color(0xFF3B82F6)     // Vibrant sky highlight
```

#### Step 2: Implement the ColorScheme Scheme in `/app/src/main/java/com/example/ui/theme/Theme.kt`

```kotlin
val AbbeyLightScheme = lightColorScheme(

    secondary = AbbeyBlueSecondary,
```

#### Step 4: Map Theme in `Theme.kt`
Configure `ShepherdTheme` to recognize the view-model state:
```kotlin
@Composable
fun ShepherdTheme(
    themePalette: String = "classic", // Loaded from DataStore preferences via ViewModel
    darkTheme: Boolean = isSystemInDarkTheme(),
    content: @Composable () -> Unit
) {
    val colors = when (themePalette) {
        "abbey_blue" -> if (darkTheme) AbbeyDarkScheme else AbbeyLightScheme
        else -> if (darkTheme) ShepherdDarkScheme else ShepherdLightScheme
    }
    
    MaterialTheme(
        colorScheme = colors,
        typography = Typography,
        content = content
    )
}
```

### Tutorial C: Adding a New Persistent Field to AppDatabase
Suppose you want to add a **"Speaker/Author"** field to individual Sermons recorded inside the application's offline organizer database schema.

#### Step 1: Update the Domain Model in `/app/src/main/java/com/example/domain/model/ShepherdModels.kt`
Edit the `Sermon` data class to include the field (give it a fallback value or make it optional):
```kotlin
data class Sermon(
    val id: String,
    val fileId: String?,
    val title: String,
    val scriptureRef: String,
    val seriesId: String?,
    val datePreached: Long?,
    val notes: String,
    val speaker: String = "Pastor" // New Field
) : Serializable
```

#### Step 2: Update the Schema Entity in `/app/src/main/java/com/example/data/local/AppDatabaseEntities.kt`
Register the column inside the `@Entity` model class and adjust domain conversion converters:
```kotlin
@Entity(tableName = "sermon")
data class SermonEntity(
    @PrimaryKey val id: String,
    val fileId: String?,
    val title: String,
    val scriptureRef: String,
    val seriesId: String?,
    val datePreached: Long?,
    val notes: String,
    val speaker: String // New DB Column
) {
    fun toDomain() = Sermon(
        id = id,
        fileId = fileId,
        title = title,
        scriptureRef = scriptureRef,
        seriesId = seriesId,
        datePreached = datePreached,
        notes = notes,
        speaker = speaker
    )

    companion object {
        fun fromDomain(sermon: Sermon) = SermonEntity(
            id = sermon.id,
            fileId = sermon.fileId,
            title = sermon.title,
            scriptureRef = sermon.scriptureRef,
            seriesId = sermon.seriesId,
            datePreached = sermon.datePreached,
            notes = sermon.notes,
            speaker = sermon.speaker
        )
    }
}
```

#### Step 3: Handle the Room Database Migration in `AppDatabase.kt`
Since you added a database field, Room must update the schema:
1. Locate check conditions or increase version number inside `AppDatabase.kt`:
```kotlin
@Database(
    entities = [
        ShepherdFileEntity::class,
        CategoryEntity::class,
        HistoryEntity::class,
        SermonEntity::class, // Entity altered
        SeriesEntity::class,
        ScriptureTagEntity::class,
        BookmarkEntity::class
    ],
    version = 2, // Increment database version number (e.g., from 1 to 2)
    exportSchema = false
)
abstract class AppDatabase : RoomDatabase() { ... }
```
2. Because Shepherd is a local companion tool, we can use destructive migrations or register a simple Migration builder inside your database instantiator method:
```kotlin
val MIGRATION_1_2 = object : Migration(1, 2) {
    override fun migrate(db: SupportSQLiteDatabase) {
        db.execSQL("ALTER TABLE sermon ADD COLUMN speaker TEXT NOT NULL DEFAULT 'Pastor'")
    }
}

// In database instantiator:
Room.databaseBuilder(context, AppDatabase::class.java, "shepherd_db")
    .addMigrations(MIGRATION_1_2)
    .fallbackToDestructiveMigration() // Last-line safety net
    .build()
```

Enable sermon writers to input their guest speaker names directly via the outline sheet interface:
```kotlin
// Input TextField state

OutlinedTextField(
```
